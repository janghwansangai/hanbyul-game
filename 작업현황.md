## 1. 실행 & 배포

```bash
cd hanbyul-game
npm install        # 최초 1회
npm run dev        # 개발서버 http://localhost:5173
npm run build      # 배포 빌드 (dist/)
```

**배포:** `git push origin main` → GitHub Actions(`.github/workflows/deploy.yml`)가 자동 빌드·배포.
- 배포가 **큐에서 안 끝나면**(GitHub 러너 혼잡, 오늘 자주 발생): 
  - `gh run rerun <id> --failed` (deploy 잡만 취소된 경우) 또는
  - 멈춘 실행 취소 후 `gh workflow run deploy.yml --ref main` 로 재트리거.
- 배포 확인: `curl -s https://janghwansangai.github.io/hanbyul-game/ | grep -oE 'index-[A-Za-z0-9_]+\.js'` 가 로컬 최신 빌드 해시와 같으면 반영 완료.

> ⚠️ **base 경로:** `vite.config.js`에서 빌드 시 base가 `/hanbyul-game/`. 로컬 dev는 `/`. 저장소 이름 바꾸면 여기도 수정.

│  └─ components/

## 3. 디버깅 꿀팁 (상태 점프)

브라우저 콘솔에서 게임 상태를 직접 바꿔 특정 장면을 바로 테스트 가능 (App.jsx에서 노출):
```js
window.__getG()            // 현재 상태 객체
const s = structuredClone(window.__getG())
s.day = 7; s.hour = 22; s.stats.popcorn = 40;
s.equipped = ['crown','sunglasses','scarf'];   // 아이템 장착 테스트
s.counters.exercise = 9;   // 운동왕 외형 테스트 (fit)
window.__setG(s)
```
- 엔딩 보기: `s.phase='ended'; s.ending='happy'` (또는 balance/normal/bad)
- 보스 보기: `s.phase='finalboss'` / `'midboss'` / 미니게임 `'arcade'`
> 주의: 프리뷰 창 리사이즈는 페이지를 리로드해 메모리 상태를 날릴 수 있음. `__getG()`가 null이면 새로 시작 후 다시 세팅.

## 4. 캐릭터 SVG 좌표 치트시트 (Character.jsx)

`viewBox="0 0 160 176"` 기준. 아이템·표정 위치 조정 시 참고:

| 부위 | 좌표 |
|---|---|
| 얼굴 | `ellipse cx80 cy78 rx48 ry46` (정수리 ~y20, 턱 ~y124) |
| 귀 | cx32 / cx128, cy82 |
| 눈 | cx58 / cx102, cy74 |
| 눈썹 | ~y50–58 |
| 코 | ~y82–88 |
| 입 | ~y97–104 |
| 앞머리(이마 덮는 선) | ~y52–56 |
| 목 | `rect x70 y110 w20 h16` |
| 몸/셔츠 | ~y116–176, x20–140 (fit은 더 넓음) |
| 가슴 별 | ~y138–155 |

**렌더 순서(중요):** 등 뒤 장착물(망토/날개) → 몸 그룹 → 귀 → 얼굴 → 머리 → 표정(눈/입) → 앞쪽 장착물(모자/왕관/안경/목도리/메달). 새 앞쪽 아이템은 파일 뒤쪽 "앞쪽 장착 아이템" 블록에 추가.

## 5. 시스템별 확장 방법

### 꾸미기 아이템 추가 (data.js + Character.jsx)
1. `data.js`의 `COSMETICS` 배열에 `{ id, icon, name, price, slot, desc }` 추가.
   - `slot`은 `head / eyes / neck / chest / back` 중 하나. 같은 슬롯은 자동으로 하나만 장착됨(`SLOT_OF`).
2. `Character.jsx`에서 `has('아이디')`로 조건부 `<g>...</g>` SVG 추가.
   - 등(back) 아이템은 "등 뒤 장착물" 블록(몸보다 위쪽), 나머지는 "앞쪽 장착 아이템" 블록에.
3. **겹침 주의:** 성장외형 기본장식과 부딪히면 숨김 조건 추가.
   - 예: 머리 아이템 있으면 운동왕 머리띠 숨김 `isFit && !has('cap') && !has('crown') && !has('headphones')`
   - 예: 선글라스 있으면 공부왕 안경 숨김 `isSmart && !has('sunglasses')`

### 성장 외형 (computeLook, Character.jsx)
- `counters.exercise / reading / totalGames` 최댓값으로 결정. max<3 또는 동점 → `normal`(균형). 값 ≥8 → level 2, ≥3 → level 1.
- 외형 종류: `fit`(운동왕) / `smart`(박사) / `messy`(거지꼴) / `normal`.
- 별명은 `LOOK_NICK`, 변신 토스트는 `GROWTH_MSG` (둘 다 Character.jsx). 토스트는 App.jsx `checkBadges()`에서 `s.lookKey`와 비교해 발생.

### 이벤트/상점/뱃지/퀴즈 숫자·문구
- 전부 `data.js`에 배열로. 밸런스 수치(행동 효과 등)는 `App.jsx`의 `applyGamePlay`/`act` 함수에.

## 6. 알려진 이슈 / 주의사항
- **GitHub Actions 큐 지연:** 오늘 러너 배정이 간헐적으로 오래 걸림. 코드 문제 아님. 재실행/재트리거로 해결.
- **프리뷰 리사이즈 → 상태 리셋:** 위 3번 주의 참고.
- **HMR stale 에러 로그:** 편집 중 남은 오래된 콘솔 에러는, 화면이 정상 렌더되면 무시. 프리뷰 서버 재시작하면 사라짐.
- **Bash 작업 폴더 리셋:** 터미널 cwd가 `levelup` 루트로 돌아갈 때 있음 → `cd hanbyul-game` 먼저.
